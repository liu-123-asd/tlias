## 🔴 第一阶段：阻塞级问题（必须立即修复）

### 1. 🔴 **安全：硬编码敏感信息**
**位置**: `application.yml:6-9`  
**问题类型**: 安全性  

**问题描述**:
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/tlias
    username: root
    password: 1234
```

**为什么这是问题**: 
- 数据库密码以明文形式存储在配置文件中
- 如果代码提交到Git，密码会泄露
- 不符合生产环境安全要求

**建议**:
```yaml
# 使用环境变量或配置中心
spring:
  datasource:
    url: ${DB_URL:jdbc:mysql://localhost:3306/tlias}
    username: ${DB_USERNAME:root}
    password: ${DB_PASSWORD:}
```

**修复优先级**: 🔴 阻塞级

### 2. 🔴 **安全：JWT密钥硬编码**
**位置**: `JwtUtils.java:14`  
**问题类型**: 安全性  

**问题描述**:
```java
private static final String BASE64_SIGN_KEY = "Zm9vYmFyZm9vYmFyZm9vYmFyZm9vYmFyZm9vYmFyZm9vYmFyZm9vYmFy";
```

**为什么这是问题**: 
- JWT签名密钥硬编码在代码中
- 一旦密钥泄露，所有令牌都可被伪造
- 无法在不同环境使用不同密钥

**建议**:
```java
// 从环境变量或配置文件读取
@Value("${jwt.secret}")
private String base64SignKey;
```

**修复优先级**: 🔴 阻塞级

### 3. 🔴 **安全：密码明文存储**
**位置**: `EmpMapper.java:52-53`  
**问题类型**: 安全性  

**问题描述**:
```java
@Select("select * from emp where username = #{username} and password = #{password}")
Emp getUsernameAndPassword(Emp emp);
```

**为什么这是问题**: 
- 密码以明文形式存储在数据库中
- 一旦数据库被攻破，所有用户密码泄露
- 不符合安全规范（应使用BCrypt等加密）

**建议**:
```java
// 1. 存储时加密
emp.setPassword(BCrypt.hashpw(emp.getPassword(), BCrypt.gensalt()));

// 2. 登录时验证
if (BCrypt.checkpw(inputPassword, storedPassword)) {
    // 登录成功
}
```

**修复优先级**: 🔴 阻塞级

### 4. 🔴 **逻辑错误：Result错误码不正确**
**位置**: `Result.java:26-29`  
**问题类型**: 逻辑缺陷  

**问题描述**:
```java
public static Result error(String msg){
    Result result = new Result();
    result.code = 1;  // ❌ 错误：应该是2
    result.msg = msg;
    return result;
}
```

**为什么这是问题**: 
- 错误响应使用了成功码（1）
- 前端无法区分成功和失败
- 会导致前端逻辑错误

**建议**:
```java
public static Result error(String msg){
    Result result = new Result();
    result.code = 2;  // ✅ 错误码应该是2
    result.msg = msg;
    return result;
}
```

**修复优先级**: 🔴 阻塞级

## 🟡 第二阶段：重要级问题（建议尽快修复）

### 6. 🟡 **性能：N+1查询问题**
**位置**: `ClazzServiceimpl.java:36-46`  
**问题类型**: 性能  

**问题描述**:
```java
for (Clazz clazz : clazzList) {
    LocalDate beginDate = clazz.getBeginDate();
    LocalDate endDate = clazz.getEndDate();
    // 在循环中计算状态
}
```

**为什么这是问题**: 
- 虽然这里只是日期计算，但如果在循环中有数据库查询，会导致N+1问题
- 应该在SQL层面一次性处理

**建议**:
```sql
-- 在SQL中使用CASE语句直接计算状态
SELECT *, 
  CASE 
    WHEN end_date < CURDATE() THEN '已结课'
    WHEN begin_date > CURDATE() THEN '未开班'
    ELSE '在读中'
  END as status
FROM clazz
```

**修复优先级**: 🟡 重要级

### 8. 🟡 **规范：字段注入而非构造器注入**
**位置**: 几乎所有Service和Controller  
**问题类型**: Spring Boot最佳实践  

**问题描述**:
```java
@Autowired  // ❌ 字段注入
private DeptService deptService;
```

**为什么这是问题**: 
- 字段注入不利于测试和不变性
- Spring官方推荐使用构造器注入

**建议**:
```java
// ✅ 构造器注入
private final DeptService deptService;

public DeptController(DeptService deptService) {
    this.deptService = deptService;
}
```

**修复优先级**: 🟡 重要级
